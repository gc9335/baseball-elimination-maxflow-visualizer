# 最大流动态可视化与扩展性能实验设计

## 1. 目标

在现有棒球淘汰 Python 项目上增加两部分能力：

1. 一个可部署到 GitHub Pages 的静态最大流教学可视化网页。
2. 一套覆盖时间、操作次数、内存、网络规模和网络密度的扩展实验。

网页参考 [graph-coloring-csp-visualizer](https://github.com/gc9335/graph-coloring-csp-visualizer) 的教学组织方式与暖纸张视觉语言，但不复制 CSP 专属的颜色域、搜索树和着色交互。

## 2. 页面设计

### 2.1 视觉方向

- 暖纸张渐变背景与细网格纹理。
- 半透明米白面板、圆角和轻阴影。
- 主色为青蓝 `#0b4f6c`，过程高亮色为橙色 `#f28f3b`。
- 标题使用衬线字体，数据、流量和伪代码使用无衬线或等宽字体。
- 不使用三维、渐变边或无意义装饰动画。

### 2.2 页面结构

页面由以下区域组成：

1. 顶部标题和关键指标卡。
2. 数据集、目标球队、算法、速度、播放控制和进度条。
3. 自适应四层残量网络主舞台：
   - 源点。
   - 比赛节点。
   - 球队节点。
   - 汇点。
4. 右侧标签页：
   - 当前步骤。
   - 伪代码。
   - 算法状态。
5. 底部可点击时间轴。
6. 性能实验图表入口。

### 2.3 自适应网络布局

- `teams4`、`teams5` 等小数据完整显示所有节点和逻辑边。
- 网络超过显示阈值时，聚合与当前事件无关的比赛节点。
- 始终展开当前 BFS/DFS 路径、瓶颈边、饱和边和最小割相关节点。
- 支持画布缩放、拖动和重置视图。
- 边标签显示 `流量 / 容量`，悬停显示剩余容量和边类型。

### 2.4 交互

- 内置数据集选择，不包含上传和在线编辑器。
- 目标球队选择。
- Edmonds–Karp / Dinic 切换。
- 播放、暂停、上一步、下一步、跳到开头、跳到结尾。
- 速度选择。
- 点击时间轴跳转。
- 点击教学标签页。
- 悬停节点和边显示解释。
- 结束时显示淘汰结论、最大流、待分配比赛数和最小割证明集合。

## 3. Python 轨迹数据

核心算法仍以 Python 为唯一事实来源。网页不重新实现最大流，而是回放 Python 导出的 JSON 轨迹。

### 3.1 通用事件

- `network-built`
- `search-start`
- `vertex-discovered`
- `edge-inspected`
- `path-found`
- `augment`
- `residual-updated`
- `search-finished`
- `min-cut`
- `completed`

### 3.2 Edmonds–Karp 专属状态

- BFS 轮次。
- 队列状态。
- 父边。
- 当前增广路径。
- 瓶颈容量。

### 3.3 Dinic 专属状态

- 分层图阶段。
- 每个节点的 level。
- DFS 调用栈。
- 当前弧指针。
- 阻塞流累计值。

### 3.4 事件快照

每个可视化事件包含：

- 事件类型和标题。
- 中文解释。
- 对应伪代码行。
- 当前最大流。
- 当前高亮节点和边。
- 全部正向逻辑边的流量、容量和剩余容量。
- 算法专属状态。
- 累计操作计数。

为避免 JSON 过大，完整边快照只在网络构建、增广、分层完成和最终状态等关键事件记录；普通检查事件仅记录变化量，导出脚本再物化为前端可直接回放的快照。

## 4. 指标设计

### 4.1 时间指标

- 总运行时间。
- 单支球队平均判定时间。
- 中位数。
- 四分位距。
- P95。
- Dinic 相对 Edmonds–Karp 加速比。

### 4.2 算法操作指标

Edmonds–Karp：

- BFS 轮次。
- 增广次数。
- 边检查次数。
- 队列入队次数。
- 反向残量边使用次数。

Dinic：

- 分层图阶段数。
- BFS 轮次。
- DFS 调用次数。
- 阻塞流推送次数。
- 边检查次数。
- 当前弧跳过次数。
- 反向残量边使用次数。

通用：

- 网络节点数。
- 逻辑边数。
- 正容量边数。
- 总容量。
- 最大流。
- 淘汰球队数。
- 直接淘汰和非直接淘汰数量。

### 4.3 内存指标

- Python `tracemalloc` 峰值内存。
- 每次全部球队判定的峰值 KiB。
- 单位网络节点峰值内存。

## 5. 实验设计

### 5.1 规模实验

固定为稠密赛程，改变球队数：

```text
8, 12, 16, 20, 24, 28
```

输出运行时间、操作次数、峰值内存和加速比。

### 5.2 密度实验

固定球队数，例如 20，改变球队对之间存在剩余比赛的概率：

```text
0.15, 0.30, 0.45, 0.60, 0.75, 0.90
```

每个密度使用多个固定种子，比较网络边数、时间和操作次数。

### 5.3 官方数据实验

对 23 个 Princeton 数据文件记录：

- 球队数。
- 比赛节点数。
- 逻辑边数。
- 淘汰数量。
- 两种算法耗时。
- 加速比。
- 峰值内存。

## 8. 测试

- 追踪开启和关闭时最大流结果一致。
- 操作计数非负且符合已知小网络。
- Edmonds–Karp 轨迹含 BFS、增广和结束事件。
- Dinic 轨迹含分层图、DFS、阻塞流和结束事件。
- 棒球网络导出节点、边和球队映射正确。
- 导出 JSON 可被解析且事件索引连续。
- 所有内置数据集均可生成网页数据。
- benchmark CSV 字段完整。
- 所有图表均生成非空 PNG。
- 浏览器验证选择器、播放控制、时间轴、标签页和网络渲染。
- 移动宽度下页面可纵向阅读，无关键控件裁切。

## 9. 部署

- 将静态站点和预生成轨迹放入 `docs/`。
- 推送到 GitHub。
- 使用 GitHub Pages 从 `main/docs` 或 Pages Actions 发布。
- 发布后验证在线 URL 可加载 `teams4`，完成播放、切换算法和显示最终证明。
